# Playscript V2 — Full Design Document
> Sport script prediction market on Somnia. Pick 5 legs. All must hit. Vault is the counterparty.

---

## 1. What Playscript V2 Is

Playscript is a sports prediction market on Somnia where users build "scripts" — picking exactly 5 legs from an 8-leg pool generated by a Somnia LLM agent for each match. All 5 must hit to win. A global LP vault is the counterparty to all scripts. Losing stakes grow the vault. Winners are paid from it.

V2 is fully agent-native:
- LLM agent generates 8 legs onchain at match creation — trustless, nobody can cherry-pick
- Reactivity drives all state transitions — no manual admin calls after creation
- JSON API agent fetches final scores from ESPN — trustless autonomous resolution

## 3. Data Structures

```solidity
enum MatchState { AWAITING_LEGS, OPEN, LOCKED, RESOLVING, SETTLED, CANCELLED }
enum Sport { Soccer, Basketball, NFL, MLB }

enum LegType { 
    HomeWin,      // 0
    AwayWin,      // 1
    Draw,         // 2
    OverGoals,    // 3 — param = threshold × 10 (e.g. 25 = over 2.5)
    UnderGoals,   // 4 — param = threshold × 10
    BothScore,    // 5 — param = min score each team must reach (0 = just score)
    CleanSheet,   // 6 — home clean sheet (away scores 0)
    Margin,       // 7 — winning margin >= param
    HTHomeWin,    // 8 — home leading at half time
    HTDraw        // 9 — draw at half time
}

struct Leg {
    LegType legType;
    uint8 weight;       // Easy=10, Medium=15, Hard=25
    uint16 param;       // e.g. 25 for "over 2.5 goals", 2 for "margin 2+"
    string description; // NOT stored onchain — emitted in event only
}

struct Match {
    uint256 espnGameId;          // ESPN event ID — used to construct resolution selectors
    string resolutionUrl;        // ESPN scoreboard URL with date e.g. ?dates=20260520
    Sport sport;
    MatchState state;
    uint64 kickoff;              // unix timestamp
    uint32 bufferSec;            // delay after match ends before resolution fires. Default per sport.
    bytes32 legsHash;            // keccak256(abi.encode(legs)) — integrity check
    uint8[8] legWeights;         // stored for payoutRate calc at lockScript and claim
    uint8[8] legTypes;           // stored for _resolveLegs()
    uint16[8] legParams;         // stored for _resolveLegs()
    uint8 resolvedLegsBitmask;   // set at resolution — 8 bits, one per leg, 1=hit 0=miss
    uint8 fetchMask;             // tracks which of 3 agent responses arrived (bits 0,1,2)
    uint256 finalHome;           // home score — set by agent callback
    uint256 finalAway;           // away score — set by agent callback
    uint256 matchLiabilityCap;   // 9.8% of vault balance AT match creation. Fixed forever.
    uint256 reactivityLockSubId;    // Reactivity Schedule ID for kickoff lock
    uint256 reactivityResolveSubId; // Reactivity Schedule ID for resolution trigger
    bool exists;
}

struct PlayerStats {
    uint32 totalScripts;
    uint32 wins;
    uint256 totalWinnings;
    uint32 rankScore;
}
```

## 5. Match State Machine

```
AWAITING_LEGS
    → [LLM agent callback handleLegsGenerated()] → OPEN

OPEN
    → [Reactivity Schedule at kickoff] → LOCKED

LOCKED
    → [Reactivity Schedule at kickoff + bufferSec] → RESOLVING

RESOLVING
    → [ESPN agent returns STATUS_FINAL, fetchMask==7] → SETTLED
    → [ESPN agent returns not final] → retry Schedule 15min → RESOLVING
    → [stuck >24h] → admin manual fallback → SETTLED

SETTLED — terminal
CANCELLED — terminal (admin only, refunds all stakes)
```

Zero admin after `createMatch()` in the happy path.

## 6. Match Creation Flow

### Step 1 — UI offchain (no gas)

UI hits ESPN to get upcoming fixtures:
```
GET https://site.api.espn.com/apis/site/v2/sports/basketball/nba/scoreboard?dates=20260520
```

Extracts:
- `events[0].id` → ESPN game ID e.g. `"401871156"`
- `events[0].date` → kickoff timestamp
- `competitors[0].team.displayName` → home team name
- `competitors[1].team.displayName` → away team name
- `competitors[0].homeAway` → confirms home/away order

Constructs resolution URL:
```
https://site.api.espn.com/apis/site/v2/sports/basketball/nba/scoreboard?dates=20260520
```

### Step 2 — createMatch() transaction

Creator calls `createMatch()` with:
- `espnGameId` — stored for selector construction at resolution
- `resolutionUrl` — stored, passed to agent at resolution time
- `kickoff` — unix timestamp
- `sport` — enum
- `bufferSec` — configurable, default 7200

```solidity
function createMatch(
    uint256 espnGameId,
    string calldata resolutionUrl,
    uint64 kickoff,
    Sport sport,
    uint32 bufferSec
) external {
    require(bufferSec >= MIN_BUFFER_SEC, "Buffer too short");
    require(kickoff > block.timestamp, "Kickoff in past");

    // vault capacity check
    uint256 vaultBal = PLAY.balanceOf(address(vault));
    uint256 matchCap = vaultBal * 98 / 1000;              // 9.8% of vault NOW
    uint256 hardFloor = vaultBal * 200 / 10000;           // 2% hard floor
    require(
        totalOutstandingLiability + matchCap <= vaultBal - hardFloor,
        "Vault full: cannot open new match"
    );

    uint256 matchId = ++matchCount;
    Match storage m = matches[matchId];
    m.espnGameId = espnGameId;
    m.resolutionUrl = resolutionUrl;
    m.sport = sport;
    m.kickoff = kickoff;
    m.bufferSec = bufferSec;
    m.matchLiabilityCap = matchCap;                       // FIXED forever at this value
    m.state = MatchState.AWAITING_LEGS;
    m.exists = true;

    matchLiabilityCap[matchId] = matchCap;
    totalOutstandingLiability += matchCap;                // reserve capacity immediately

    // register Reactivity Schedule 1 — kickoff lock
    m.reactivityLockSubId = REACTIVITY.registerSchedule(
        kickoff,
        address(this),
        abi.encode(matchId, SCHEDULE_LOCK)
    );

    // register Reactivity Schedule 2 — resolution trigger
    m.reactivityResolveSubId = REACTIVITY.registerSchedule(
        kickoff + bufferSec,
        address(this),
        abi.encode(matchId, SCHEDULE_RESOLVE)
    );

    // call LLM agent to generate 8 legs
    _requestLegsGeneration(matchId, homeTeamName, awayTeamName, sport);

    emit MatchCreated(matchId, espnGameId, kickoff, sport);
}
```

### Step 3 — LLM agent generates legs onchain

```solidity
function _requestLegsGeneration(
    uint256 matchId,
    string memory home,
    string memory away,
    Sport sport
) internal {
    bytes memory prompt = abi.encodePacked(
        "Generate exactly 8 prediction legs for ", _sportName(sport), " match: ",
        home, " (home) vs ", away, " (away). ",
        "Each leg verifiable from final score and half-time score only. ",
        "No player stats. Return JSON array only: ",
        '[{"id":0,"description":"...","legType":"HomeWin","weight":10,"param":0},...]',
        " Difficulty weights: Easy=10, Medium=15, Hard=25. ",
        "Include variety: result, goals, margin, half-time."
    );

    uint256 reqId = PLATFORM.createRequest{value: LLM_DEPOSIT}(
        LLM_AGENT_ID,
        address(this),
        this.handleLegsGenerated.selector,
        prompt
    );

    pendingLegsRequest[reqId] = matchId;
}
```

Validators all run same LLM, same seed, same prompt. All return same JSON. Consensus reached. Nobody can manipulate legs.

### Step 4 — handleLegsGenerated() callback

```solidity
function handleLegsGenerated(
    uint256 requestId,
    Response[] memory responses,
    ResponseStatus status
) external {
    require(msg.sender == address(PLATFORM));

    uint256 matchId = pendingLegsRequest[requestId];
    delete pendingLegsRequest[requestId];

    Match storage m = matches[matchId];

    // parse JSON into Leg structs
    Leg[8] memory legs = _parseLegsJSON(responses[0].result);

    // store integrity hash
    m.legsHash = keccak256(abi.encode(legs));

    // store only what resolution and payout need onchain
    for (uint8 i = 0; i < 8; i++) {
        m.legWeights[i] = legs[i].weight;
        m.legTypes[i]   = uint8(legs[i].legType);
        m.legParams[i]  = legs[i].param;
    }

    // emit full legs including descriptions — UI reads from here
    emit LegsGenerated(matchId, m.legsHash, legs);

    // now open for scripts
    m.state = MatchState.OPEN;
    emit MatchOpen(matchId);
}
```

**What's stored onchain per leg:** `weight (uint8)`, `legType (uint8)`, `param (uint16)` — 4 bytes per leg, 32 bytes total for all 8 legs. Plus `legsHash (bytes32)`.

**What's NOT stored onchain:** descriptions. UI reads them from the `LegsGenerated` event.

## 8. _resolveLegs() — Exact Logic

```solidity
function _checkLeg(uint8 legType, uint16 param, uint8 h, uint8 a) 
    internal pure returns (bool) 
{
    if (legType == 0) return h > a;                              // HomeWin
    if (legType == 1) return a > h;                              // AwayWin
    if (legType == 2) return h == a;                             // Draw
    if (legType == 3) return uint16(h + a) * 10 > param;        // OverGoals: (h+a)*10 > param
    if (legType == 4) return uint16(h + a) * 10 < param;        // UnderGoals
    if (legType == 5) return h >= uint8(param) && a >= uint8(param); // BothScore
    if (legType == 6) return h > 0 && a == 0;                   // CleanSheet (home)
    if (legType == 7) {
        uint8 winner = h > a ? h - a : a - h;
        return winner >= uint8(param);                           // Margin
    }
    if (legType == 8) return hHalf > aHalf;                     // HTHomeWin (needs half scores)
    if (legType == 9) return hHalf == aHalf;                    // HTDraw
    return false;
}

function _resolveLegs(uint256 matchId) internal {
    Match storage m = matches[matchId];
    uint8 h = uint8(m.finalHome);
    uint8 a = uint8(m.finalAway);
    uint8 resolved = 0;

    for (uint8 i = 0; i < 8; i++) {
        if (_checkLeg(m.legTypes[i], m.legParams[i], h, a)) {
            resolved |= (uint8(1) << i);
        }
    }
    m.resolvedLegsBitmask = resolved;
}
```

**Example: Spurs 126, Timberwolves 97**

```
leg 0: HomeWin → 126 > 97 = true  → bit 0 = 1
leg 1: AwayWin → 97 > 126 = false → bit 1 = 0
leg 2: Draw → 126 == 97 = false   → bit 2 = 0
leg 3: Over220 → (126+97)=223 > 220 = true → bit 3 = 1
leg 4: Under200 → 223 < 200 = false → bit 4 = 0
leg 5: BothScore100 → 126>=100 && 97>=100 = true (97 just hits) → bit 5 = 1

wait: 97 >= 100 = false → bit 5 = 0

leg 6: Margin10 → (126-97)=29 >= 10 = true → bit 6 = 1
leg 7: Margin20 → 29 >= 20 = true → bit 7 = 1

resolved = 0b11001001 = 0xC9
bits set: 0, 3, 6, 7
only 4 legs hit — nobody wins this match
losing stakes stay in vault, LPs profit
```

## 9. lockScript — Full Math

```solidity
function lockScript(
    uint256 matchId, 
    uint8 legBitmask, 
    uint256 requestedStake
) external {
    Match storage m = matches[matchId];
    require(m.state == MatchState.OPEN,          "Not open");
    require(block.timestamp < m.kickoff,          "Kickoff passed");
    require(_popcount(legBitmask) == 5,           "Must pick exactly 5 legs");

    // Step 1: compute difficulty score
    uint256 score = 0;
    for (uint8 i = 0; i < 8; i++) {
        if (legBitmask & (uint8(1) << i) != 0) {
            score += m.legWeights[i];
        }
    }
    // score range: 50 (5x Easy) to 125 (5x Hard)

    // Step 2: get payout rate
    uint256 payoutRate = _getPayoutRate(score);
    // e.g. score=80 → payoutRate=70 meaning 7.0x

    // Step 3: compute requested liability
    uint256 requestedLiability = requestedStake * payoutRate / 10;
    // e.g. stake=100, rate=70 → liability=700

    // Step 4: compute available room (BOTH checks must pass)
    uint256 matchRoom = matchLiabilityCap[matchId] - matchLiability[matchId];
    uint256 vaultBal = PLAY.balanceOf(address(vault));
    uint256 hardFloor = vaultBal * 200 / 10000;           // 2% of vault
    uint256 globalRoom = vaultBal - totalOutstandingLiability - hardFloor;
    uint256 availableRoom = matchRoom < globalRoom ? matchRoom : globalRoom;
    // takes the TIGHTER of match cap and global vault capacity

    // Step 5: partial fill if needed
    uint256 actualLiability = requestedLiability > availableRoom
        ? availableRoom : requestedLiability;
    uint256 actualStake = actualLiability * 10 / payoutRate;
    // e.g. only 500 room → actualLiability=500 → actualStake=500*10/70=71

    // Step 6: pull funds, refund excess
    PLAY.transferFrom(msg.sender, address(this), requestedStake);
    if (requestedStake > actualStake) {
        PLAY.transfer(msg.sender, requestedStake - actualStake); // refund
    }

    // Step 7: fee and vault transfer
    uint256 fee = actualStake * 5 / 1000;                 // 0.5% fee
    uint256 netStake = actualStake - fee;
    PLAY.transfer(address(vault), actualStake);
    // full actualStake goes to vault — fee grows LP value via vault appreciation

    // Step 8: update liability tracking
    matchLiability[matchId] += actualLiability;
    // NOTE: totalOutstandingLiability was already incremented by matchCap at creation
    // it does NOT increment again here — matchLiability fills up toward the cap

    // Step 9: update leg inclusion counts
    for (uint8 i = 0; i < 8; i++) {
        if (legBitmask & (uint8(1) << i) != 0) {
            legInclusionCount[matchId][i]++;
        }
    }

    // Step 10: mint ERC-1155
    // tokenId is deterministic — same legs on same match = same tokenId
    uint256 tokenId = uint256(keccak256(abi.encodePacked(matchId, legBitmask)));
    _mint(msg.sender, tokenId, netStake, "");
    // balance of tokenId = netStake amount (18 decimals)

    if (tokenOriginalMinter[tokenId] == address(0)) {
        tokenOriginalMinter[tokenId] = msg.sender;
    }

    emit ScriptLocked(matchId, legBitmask, msg.sender, actualStake, netStake, payoutRate);
}
```

## 10. Payout Rate Table

Difficulty score = sum of weights of 5 selected legs.

```solidity
function _getPayoutRate(uint256 score) internal pure returns (uint256) {
    if (score <= 50)  return 18;   // 1.8x — 5x Easy
    if (score <= 55)  return 22;   // 2.2x
    if (score <= 60)  return 28;   // 2.8x
    if (score <= 65)  return 35;   // 3.5x
    if (score <= 70)  return 45;   // 4.5x
    if (score <= 75)  return 55;   // 5.5x
    if (score <= 80)  return 70;   // 7.0x
    if (score <= 90)  return 90;   // 9.0x
    if (score <= 100) return 120;  // 12.0x
    if (score <= 110) return 150;  // 15.0x
    if (score <= 120) return 180;  // 18.0x
    return 200;                    // 20.0x — 5x Hard
}
// payoutRate is ×10 integer: rate 70 means 7.0x payout
// payout = netStake * payoutRate / 10
```

**Score examples:**
```
5x Easy:            10+10+10+10+10 = 50  → 1.8x
3x Easy + 2x Medium: 10+10+10+15+15 = 60 → 2.8x
2x Easy + 3x Medium: 10+10+15+15+15 = 65 → 3.5x
1x Easy + 3x Medium + 1x Hard: 10+15+15+15+25 = 80 → 7.0x
2x Medium + 3x Hard: 15+15+25+25+25 = 105 → 12.0x (falls in <=110)
5x Hard:            25+25+25+25+25 = 125 → 20.0x
```

## 11. Reactivity — Kickoff Lock

```solidity
uint8 constant SCHEDULE_LOCK    = 1;

function _onEvent(EventContext memory ctx) internal override {
    (uint256 matchId, uint8 scheduleType) = abi.decode(ctx.data, (uint256, uint8));
```

## 12. Resolution — JSON API Agent

```solidity
uint8 constant FIELD_HOME   = 0;
uint8 constant FIELD_AWAY   = 1;
uint8 constant FIELD_STATUS = 2;

struct Resolution {
    uint256 matchId;
    uint8 field;
}

function _triggerResolution(uint256 matchId) internal {
    matches[matchId].state = MatchState.RESOLVING;

    Match storage m = matches[matchId];
    string memory gameId = Strings.toString(m.espnGameId);

    // construct JSON selectors using stored espnGameId
    string memory homeSelector = string(abi.encodePacked(
        'events.#(id=="', gameId, '").competitions.0.competitors.#(homeAway=="home").score'
    ));
    string memory awaySelector = string(abi.encodePacked(
        'events.#(id=="', gameId, '").competitions.0.competitors.#(homeAway=="away").score'
    ));
    string memory statusSelector = string(abi.encodePacked(
        'events.#(id=="', gameId, '").status.type.name'
    ));

    uint256 deposit = AGENT_DEPOSIT;

    uint256 reqHome = PLATFORM.createRequest{value: deposit}(
        JSON_API_AGENT_ID,
        address(this),
        this.handleResolution.selector,
        abi.encodeWithSelector(
            IJsonApiAgent.fetchUint.selector,
            m.resolutionUrl,
            homeSelector,
            uint8(0)
        )
    );

    uint256 reqAway = PLATFORM.createRequest{value: deposit}(
        JSON_API_AGENT_ID,
        address(this),
        this.handleResolution.selector,
        abi.encodeWithSelector(
            IJsonApiAgent.fetchUint.selector,
            m.resolutionUrl,
            awaySelector,
            uint8(0)
        )
    );

    uint256 reqStatus = PLATFORM.createRequest{value: deposit}(
        JSON_API_AGENT_ID,
        address(this),
        this.handleResolution.selector,
        abi.encodeWithSelector(
            IJsonApiAgent.fetchString.selector,
            m.resolutionUrl,
            statusSelector
        )
    );

    pendingResolution[reqHome]   = Resolution(matchId, FIELD_HOME);
    pendingResolution[reqAway]   = Resolution(matchId, FIELD_AWAY);
    pendingResolution[reqStatus] = Resolution(matchId, FIELD_STATUS);
}
```

### handleResolution() callback

```solidity
function handleResolution(
    uint256 requestId,
    Response[] memory responses,
    ResponseStatus status
) external {
    require(msg.sender == address(PLATFORM));

    Resolution memory r = pendingResolution[requestId];
    delete pendingResolution[requestId];
    Match storage m = matches[r.matchId];

    if (r.field == FIELD_HOME) {
        m.finalHome = abi.decode(responses[0].result, (uint256));
        m.fetchMask |= 1;   // bit 0
    } else if (r.field == FIELD_AWAY) {
        m.finalAway = abi.decode(responses[0].result, (uint256));
        m.fetchMask |= 2;   // bit 1
    } else if (r.field == FIELD_STATUS) {
        string memory s = abi.decode(responses[0].result, (string));
        if (!_isFinished(s)) {
            // not STATUS_FINAL — schedule retry in 15 minutes
            REACTIVITY.registerSchedule(
                block.timestamp + 900,
                address(this),
                abi.encode(r.matchId, SCHEDULE_RESOLVE)
            );
            return;
        }
        m.fetchMask |= 4;   // bit 2
    }

    // fetchMask = 0b111 = 7 means all three responses received
    if (m.fetchMask == 7) {
        _resolveLegs(r.matchId);
        _settleMatch(r.matchId);
    }
}

function _isFinished(string memory s) internal pure returns (bool) {
    return keccak256(bytes(s)) == keccak256(bytes("STATUS_FINAL"));
}

function _settleMatch(uint256 matchId) internal {
    // clear liability immediately — LPs can withdraw now
    // matchLiability[matchId] fills toward matchCap but cap was already
    // counted in totalOutstandingLiability at creation
    // actual liability = matchLiability[matchId] (what was actually staked × rates)
    totalOutstandingLiability -= matchLiabilityCap[matchId];
    // add back unused cap capacity
    uint256 unusedCap = matchLiabilityCap[matchId] - matchLiability[matchId];
    // totalOutstandingLiability already cleared — vault free float increases immediately

    matchLiability[matchId] = 0;
    matches[matchId].state = MatchState.SETTLED;
    emit MatchSettled(matchId, matches[matchId].resolvedLegsBitmask);
}
```

## 13. Vault — Liability and Free Float Math

```solidity
// PlayVault.sol

uint256 public totalOutstandingLiability;
uint256 public constant HARD_FLOOR_BPS = 200; // 2%

function freeFloat() public view returns (uint256) {
    uint256 bal = PLAY.balanceOf(address(this));
    uint256 floor = bal * HARD_FLOOR_BPS / 10000;
    uint256 available = bal - totalOutstandingLiability - floor;
    return available > 0 ? available : 0;
}
```

**How free float changes:**

| Event | Effect on freeFloat |
|---|---|
| LP deposits PLAY | bal ↑ → freeFloat ↑ |
| LP withdraws PLAY | bal ↓ → freeFloat ↓ |
| Match created (9.8% reserved) | totalOutstandingLiability ↑ → freeFloat ↓ |
| Match resolves | totalOutstandingLiability ↓ → freeFloat ↑ immediately |
| Winner claims | vault.pay() → bal ↓ → freeFloat ↓ |
| Losing stakes (no winner) | bal stays in vault → LPs profit via share appreciation |

**9.8% cap math:**

```
Vault = 1,000,000 PLAY
hardFloor = 20,000 PLAY (2%)
matchCap per match = 98,000 PLAY (9.8%)

Match 1 created:
  matchLiabilityCap[1] = 98,000
  totalOutstandingLiability = 98,000
  freeFloat = 1,000,000 - 98,000 - 20,000 = 882,000

Match 2 created:
  matchLiabilityCap[2] = 98,000 (still 9.8% of 1,000,000 — vault balance unchanged)
  totalOutstandingLiability = 196,000
  freeFloat = 1,000,000 - 196,000 - 20,000 = 784,000

10 matches created:
  totalOutstandingLiability = 980,000
  freeFloat = 1,000,000 - 980,000 - 20,000 = 0

11th match attempted:
  require(980,000 + 98,000 <= 1,000,000 - 20,000)
  require(1,078,000 <= 980,000) → FAILS → BLOCKED
```

Max ~10 concurrent full-capacity matches from any vault size.

**9.8% is always from total vault balance at creation — NOT from remaining free float:**
```
Match 1 cap = 9.8% of 1,000,000 = 98,000
Match 2 cap = 9.8% of 1,000,000 = 98,000  ← same total, not 9.8% of 902,000
```

The guard check (totalOutstandingLiability + newCap <= vaultBal - hardFloor) is what prevents over-commitment — not a shrinking percentage base.

## 14. LP Deposit and Withdrawal

```solidity
function deposit(uint256 amount) external {
    uint256 vaultBal = PLAY.balanceOf(address(this));
    uint256 shares = vaultBal == 0
        ? amount
        : amount * LP_PLAY.totalSupply() / vaultBal;
    PLAY.transferFrom(msg.sender, address(this), amount);
    LP_PLAY.mint(msg.sender, shares);
    emit Deposited(msg.sender, amount, shares);
}

function withdraw(uint256 lpAmount) external {
    uint256 vaultBal = PLAY.balanceOf(address(this));
    uint256 userEntitlement = lpAmount * vaultBal / LP_PLAY.totalSupply();
    uint256 available = freeFloat();
    uint256 actualPayout = userEntitlement > available ? available : userEntitlement;
    uint256 actualLpBurn = actualPayout * LP_PLAY.totalSupply() / vaultBal;
    LP_PLAY.burn(msg.sender, actualLpBurn);
    PLAY.transfer(msg.sender, actualPayout);
    // remainder of entitlement stays — LP comes back as matches resolve
    emit Withdrawn(msg.sender, actualPayout, actualLpBurn);
}
```

LP withdrawal is gated by free float only. No lockup. As matches resolve, liability clears, free float increases, LPs can withdraw more.

## 15. Claim

```solidity
function claim(uint256 matchId, uint8 legBitmask, uint256 amount) external {
    Match storage m = matches[matchId];
    require(m.state == MatchState.SETTLED, "Not settled");

    // all 5 picked legs must be set in resolvedLegsBitmask
    require(
        (legBitmask & m.resolvedLegsBitmask) == legBitmask,
        "Not a winner"
    );

    // derive tokenId — same as mint
    uint256 tokenId = uint256(keccak256(abi.encodePacked(matchId, legBitmask)));

    // burn ERC-1155 balance
    _burn(msg.sender, tokenId, amount);

    // compute payout
    uint256 score = 0;
    for (uint8 i = 0; i < 8; i++) {
        if (legBitmask & (uint8(1) << i) != 0) {
            score += m.legWeights[i];
        }
    }
    uint256 payoutRate = _getPayoutRate(score);
    uint256 payout = amount * payoutRate / 10;

    // vault pays winner — vault balance decreases here
    vault.pay(msg.sender, payout);

    // update leaderboard using original minter
    address original = tokenOriginalMinter[tokenId];
    playerStats[original].wins++;
    playerStats[original].totalWinnings += payout;
    playerStats[original].rankScore = uint32(
        playerStats[original].wins * 100 +
        playerStats[original].totalWinnings / 1e18 * 10
    );

    emit ScriptClaimed(matchId, legBitmask, msg.sender, payout);
}
```

**Win check explained:**
```
legBitmask = user's picked legs = 0b00001001 = legs [0, 3]... wait needs 5 bits

example: user picked legs [0,3,5,6,7] → bitmask = 0b11101001 = 0xE9
resolvedLegsBitmask from example above = 0b11001001 = 0xC9

(0xE9 & 0xC9) = 0b11001001 = 0xC9
0xC9 != 0xE9 → NOT A WINNER (leg 5 didn't hit — bit 5 is 0 in resolved)

user picked legs [0,3,6,7] + one more that hit... but only 4 hit total in this match
so nobody wins — no valid 5-bit subset of 0xC9 exists because 0xC9 only has 4 bits set
```

UI enumerates all C(8,5) = 56 combinations client-side, checks each against `resolvedLegsBitmask` instantly. No onchain iteration.

## 20. Key Invariants — Never Break These

1. `totalOutstandingLiability` incremented at `createMatch()` by `matchCap`, cleared at `_settleMatch()` by `matchCap`. Never incremented at `lockScript()`.
2. `matchLiability[matchId]` incremented at `lockScript()`, cleared at `_settleMatch()`. Always ≤ `matchLiabilityCap[matchId]`.
3. `matchLiabilityCap[matchId]` = 9.8% of vault balance AT creation time. Never changes after that.
4. `freeFloat()` = `vaultBal - totalOutstandingLiability - hardFloor`. Always ≥ 0 enforced by guards.
5. `tokenId = keccak256(matchId, legBitmask)` — same at mint and burn. Never computed differently.
6. `resolvedLegsBitmask` written exactly once at `_resolveLegs()`. Never written again.
7. `payoutRate` computed from stored `legWeights` — same function at `lockScript()` and `claim()`. Never stored separately.
8. LP withdrawal never exceeds `freeFloat()`. Partial withdrawal returns proportional LP shares burned.
9. State transitions are one-way. No state can go backwards.
10. Only `PLATFORM` address can call `handleResolution()` and `handleLegsGenerated()`.
